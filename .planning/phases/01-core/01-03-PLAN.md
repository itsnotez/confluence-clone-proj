PermissionService.java (@Service @RequiredArgsConstructor):

권한 레벨 우선순위 (높을수록 강함): SPACE_ADMIN > WRITE > READ > NONE
판단 우선순위: 개인 권한(USER) > 그룹 권한(GROUP) > 전체 권한(ALL)

핵심 메서드:

String resolveSpacePermission(Long spaceId, Long userId, List<Long> userGroupIds):
  1. USER 타입으로 개인 권한 조회 → 있으면 반환
  2. GROUP 타입으로 사용자의 그룹 권한 조회 → 복수일 경우 가장 높은 권한 반환
  3. ALL 타입 조회 → 있으면 반환
  4. 없으면 "NONE"

boolean canRead(Long spaceId, Long userId, String userRole, List<Long> userGroupIds):
  - userRole == "SITE_ADMIN" → true
  - resolveSpacePermission 결과가 READ/WRITE/SPACE_ADMIN 이상이면 true

boolean canWrite(Long spaceId, Long userId, String userRole, List<Long> userGroupIds):
  - userRole == "SITE_ADMIN" → true
  - WRITE/SPACE_ADMIN 이상이면 true

boolean isSpaceAdmin(Long spaceId, Long userId, String userRole, List<Long> userGroupIds):
  - userRole == "SITE_ADMIN" → true
  - SPACE_ADMIN 이면 true

String resolveContentPermission(Long contentId, Long spaceId, Long userId, String userRole, List<Long> userGroupIds):
  - ContentPermission이 있으면 동일한 우선순위로 판단
  - 없으면 resolveSpacePermission 결과 상속 (콘텐츠 권한 부재 시 Space 권한 사용)

void grantSpacePermission(Long spaceId, String subjectType, Long subjectId, String permissionLevel):
  - 기존 권한이 있으면 permissionLevel 업데이트, 없으면 신규 생성

void revokeSpacePermission(Long spaceId, String subjectType, Long subjectId):
  - 권한 레코드 삭제

List<SpacePermission> findSpacePermissions(Long spaceId):
  - findBySpaceId 반환

단순화를 위해 GroupMemberRepository를 주입받아 사용자 그룹 ID 목록을 조회하는 헬퍼 메서드 포함:
List<Long> getUserGroupIds(Long userId) — findByUserId(userId).stream().map(gm -> gm.getGroup().getId()).toList()
(단, GroupMemberRepository는 Plan 01에서 생성 — 이 Plan은 Wave 1에서 독립 실행이므로 GroupMemberRepository 없이도 컴파일되어야 함)

SpacePermissionController.java (@RestController @RequestMapping("/spaces/{spaceKey}/permissions")):
- GET /spaces/{spaceKey}/permissions → findSpacePermissions(spaceId)
  → Space를 SpaceRepository에서 조회, spaceId 추출
