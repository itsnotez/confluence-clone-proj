ContentService.java (@Service @Transactional @RequiredArgsConstructor):

List<ContentDto.TreeNode> getContentTree(Long spaceId, Long currentUserId, String userRole):
  - findBySpaceIdAndDeletedAtIsNull(spaceId)로 전체 목록 조회
  - 메모리에서 parentId 기준으로 트리 구성 (재귀 없이 Map<parentId, List<Content>> 구성 후 buildTree)
  - canRead 체크 후 응답

ContentDto.Response getContent(Long contentId, Long currentUserId, String userRole):
  - findByIdAndDeletedAtIsNull → CONTENT_NOT_FOUND
  - PermissionService.canRead 체크 → PERMISSION_DENIED
  - 현재 버전 body를 Response에 포함

ContentDto.Response createContent(String spaceKey, ContentDto.CreateRequest req, Long createdById, String userRole):
  - SpaceRepository로 spaceId 조회
  - PermissionService.canWrite 체크
  - Content 저장 (status="DRAFT")
  - body가 있으면 ContentVersion(versionNo=1, body=req.body) 저장 후 content.currentVersionId 업데이트

ContentDto.Response updateContent(Long contentId, ContentDto.UpdateRequest req, Long userId, String userRole):
  - Content 조회 → canWrite 체크
  - title/body 업데이트 (Draft 덮어쓰기: currentVersionId의 body 업데이트)

ContentDto.Response publishContent(Long contentId, ContentDto.PublishRequest req, Long userId, String userRole):
  - Content 조회 → canWrite 체크
  - 새 versionNo = countByContentId + 1
  - ContentVersion 신규 생성
  - content.currentVersionId 업데이트, status="PUBLISHED"

void deleteContent(Long contentId, Long userId, String userRole):
  - canWrite 체크
  - content.deletedAt = now(), status = "ARCHIVED"

List<ContentDto.VersionResponse> getVersions(Long contentId, Long userId, String userRole):
  - canRead 체크
  - findByContentIdOrderByVersionNoDesc
